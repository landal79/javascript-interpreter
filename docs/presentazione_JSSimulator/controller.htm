Simulator (Controller)

  • Questo componente rappresenta il controller dell'applicazione, in pratica è il componente che crea tutte le altri parti dell'applicazione;
  • La classe Simulator, classe main dell'applicazione, specializza la classe IODevice, in pratica configura l'interfaccia di I/O come una finestra con un dispositivo di input uno di output e due pulsanti azione;
  • L'azione nel dispositivo di I/O è stata impostata attraverso il pattern Command, infatti le due azioni corrispondenti ai pulsanti, implementano l'interfaccia ICommand; inoltre dato che le due azioni hanno molti passi comuni, è stato usato il pattern template method: è stata creata una classe Command, che nell'implementazione di execute invoca un metodo astratto che verrà specializzato dalle sottoclassi.
