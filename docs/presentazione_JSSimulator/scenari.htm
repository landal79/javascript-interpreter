Desctizione casi d'uso

Interpretazione Albero Analisi Sintattica Analisi Lessicale

Caso d'uso: Interpretazione

Descrizione:
  • compie l'interpretazione (secondo le regole semantiche stabilite) di un programma scritto in javascript di lunghezza qualsiasi, ricevuto in ingresso attraverso il dispositivo di input.

 
Attore Primario:
  • Utente

 
Precondizioni:
  • è disponibile una frase di ingresso sul dispositivo di input;

 

Postcondizioni:
  • viene visualizzata in uscita sul dispositivo di output l'elenco delle variabili, contenenti i valori dopo la valutazione della frase;

 
Scenario Principale:
  UTENTE SISTEMA
1 Richiede l'interpretazione del programma  
2   reperisce la frase dal dispositivo di input
3   include ( Analisi Lessicale )
4   include ( Analisi Sintattica )
5   esegue l'interpretazione della frase
6   fornisce in uscita l'elenco delle variabili
I scenario di errore:
2.1 SISTEMA: manca la frase sul disposistivo di input, la valutazione viene interrota, viene visualizzato un messaggio di errore;
II scenario di errore:
3.1 SISTEMA: errore lessicale, termina la valutazione con insuccesso, viene visualizzato un messaggio di errore;
III scenario di errore:
4.1 SISTEMA: errore sintattico, termina la valutazione con insuccesso, viene visualizzato un messaggio di errore;
IV scenario di errore:
5.1 SISTEMA: errore durante la valutazione, termina la valutazione con insuccesso, viene visualizzato un messaggio di errore;

Caso d'uso: Albero

Descrizione:
  • viene visualizzata la rappresentazione sotto forma di albero di derivazione di una frase presente o di un breve programma sul dispositivo di output. Si è scelto di rappresentare l'albero con la notazione XML.

 
Attore Primario:
  • Utente

 
Precondizioni:
  • è disponibile una singola frase di ingresso sul dispositivo di input;

 

Postcondizioni:
  • viene visualizzata in uscita sul dispositivo di output una rappresentazione grafica della frase;

 
Scenario Principale:
  UTENTE SISTEMA
1 Richiede la visualizzazione della frase sotto forma di albero  
2   reperisce la frase dal dispositivo di input
3   include ( Analisi Lessicale )
4   include ( Analisi Sintattica )
5   fornisce la rappresentazione sotto forma di albero della frase sul dispositivo di output;
I scenario di errore:
2.1 SISTEMA: manca la frase sul disposistivo di input, la valutazione viene interrota, viene visualizzato un messaggio di errore;
II scenario di errore:
3.1 SISTEMA: errore lessicale, termina la valutazione con insuccesso, viene visualizzato un messaggio di errore;
III scenario di errore:
4.1 SISTEMA: errore sintattico, termina la valutazione con insuccesso, viene visualizzato un messaggio di errore;

Caso d'uso: Analisi Sintattica

Descrizione:
  • viene compiuta l'analisi sintattica di un programma javascript;

 
Osservazioni:
  • l'analisi sintattica può essere invocata direttamente dall'utente, oppure può essere il sistema che la invoca per eseguire altri casi d'uso, in cui si richiede l'analisi sintattica;

 

Attore Primario:
  • Utente

 
Precondizioni:
  • è disponibile una singola frase di ingresso sul dispositivo di input;

 

Postcondizioni:
  • la frase è sintatticamente corretta;

 
Scenario Principale:
  UTENTE SISTEMA
1 Richiede l'analisi sintattica  
2   reperisce la frase dal dispositivo di input
3   include ( Analisi Lessicale )
4   esegue l'analisi sintattica
5   fornisce in uscita un messaggio che indica la correttezza sintattica della frase.
Scenario alternativo:
  SISTEMA
3 include ( Analisi Lessicale )
4 esegue l'analisi sintattica
  • quando il caso d'uso viene richiesto dal sistema vengono compiuti solo i passi 3 e 4 dello scenario principale;
I scenario di errore:
2.1 SISTEMA: manca la frase sul disposistivo di input, la valutazione viene interrota, viene visualizzato un messaggio di errore;
II scenario di errore:
3.1 SISTEMA: errore lessicale, termina la valutazione con insuccesso, viene visualizzato un messaggio di errore;
III scenario di errore:
4.1 SISTEMA: errore sintattico, termina la valutazione con insuccesso, viene visualizzato un messaggio di errore;

Caso d'uso: Analisi Lessicale

Descrizione:
  • viene compiuta l'analisi lessicale di un programma javascript;

 
Precondizioni:
  • la frase da analizzare è già disponibile;

 

Postcondizioni:
  • la frase è sintatticamente corretta;

 
Scenario Principale:
  SISTEMA
1 reperisce la frase da analizzare
2 esegue l'analisi lessicale
I scenario di errore:
1.1 SISTEMA: non è disponibile nessuna frase, la valutazione viene interrota, viene visualizzato un messaggio di errore;
II scenario di errore:
2.1 SISTEMA: errore lessicale, termina la valutazione con insuccesso, viene visualizzato un messaggio di errore;
